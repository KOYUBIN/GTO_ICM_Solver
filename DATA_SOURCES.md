# 데이터 출처 및 산출 방식 (Data Provenance)

이 문서는 GTO/ICM Solver가 사용하는 **모든** 전략 데이터가 어디서 왔고 어떻게
만들어졌는지를 있는 그대로 기록합니다. 핵심 요약: **외부 상용 데이터는 일절
사용하지 않으며**, 모든 수치는 이 저장소 안의 코드(수학 공식, 몬테카를로
시뮬레이션, 휴리스틱)로 자체 산출한 것입니다. 상당수는 "근사치"이며, 각 항목에
그 한계를 명시합니다.

---

## 1. 프리플랍 차트 — 수작업 근사 + 휴리스틱

- **위치**: `packages/engine/src/charts.ts`
- **내용**: 100bb 6-max RFI(오픈 레이즈) 및 주요 vs-RFI(콜/3벳) 스팟.
- **출처**: 공개적으로 널리 알려진 100bb 6-max 솔루션의 **일반적인 형태를
  수작업으로 근사해 레인지 표기법("22+, A2s+, …")으로 직접 인코딩**한
  것입니다. 특정 상용 솔버의 출력물을 복사한 것이 아닙니다.
- **폴백**: 저장된 차트가 없는 스팟은 Chen 공식 핸드 강도 기반의
  휴리스틱(포지션·스택 깊이에 따라 상위 N% 오픈)으로 대체되며, 결과에
  `source: 'heuristic'`로 표시됩니다.
- **한계**: 솔버 정밀도가 아닌 "실전 참고용 근사"입니다. 혼합 빈도(`:0.5` 등)는
  일부 스팟에만 있습니다.

## 2. 자체 시뮬레이션 RFI 데이터 — 본 저장소의 셀프 시뮬 파이프라인

- **위치**: 데이터 `packages/engine/src/generated/rfi-sim.json`,
  생성 스크립트 `packages/engine/scripts/gen-preflop.mjs`,
  소비 API `packages/engine/src/simdata.ts` (`simRfiRange(pos, stackBB)`).
- **재생성**: `npm run build:engine && npm run gen:preflop -w @gto/engine`
  (블록 단위 체크포인트로 중단 후 재개 가능, 시드 고정으로 결과 재현 가능).

### 모델

- 6-max 오픈 포지션 5곳(UTG/MP/CO/BTN/SB) × 스택 깊이 3종(20/50/100bb) ×
  **169개 스타팅 핸드 전부**에 대해 엔진의 `openRaiseEv`(몬테카를로 칩-EV
  모델)를 계산합니다. 레이즈 사이즈 2.3bb, 데드머니 1.5bb.
- 상대의 컨티뉴 레인지는 포지션별로 키잉된 상위 X% 레인지
  (`topPercentRange`, Chen 강도 순): UTG 17% / MP 18% / CO 24% / BTN 31% /
  SB 52% (수비자 1명 기준, 남은 인원수만큼 폴드 확률 (1-X)^n 적용).
  얕은 스택에서는 컨티뉴가 3벳-올인 위주가 되는 것을 근사하기 위해 X를 약간
  넓히고 실현율을 소폭 낮춥니다.
- 에퀴티 실현율(realization)은 포지션 기본값(`realizationFor`)에 Chen 점수
  기반 핸드 품질 계수를 곱해 사용합니다(프리미엄 핸드는 에퀴티를 거의 다
  실현, 쓰레기 핸드는 일부만 실현). 이 계수는 **캘리브레이션된 휴리스틱**입니다.
- **RFI 레인지 도출 규칙**: 시뮬 EV > 0 인 핸드만 오픈.

### 시뮬레이션 볼륨 (실제 실행 기록)

- 핸드당 **20,000회** 몬테카를로 보드 롤아웃 × 2,535개 핸드 평가
  (5 포지션 × 3 스택 × 169 핸드) = **총 50,700,000회 롤아웃**, 실행 시간 약
  **110초** (단일 Node 프로세스). 메타데이터는 JSON의 `meta`
  (`iterationsPerHand`, `totalIterations`, `generatedAt`, `configHash`)에
  기록됩니다.

### 한계 (중요)

- **칩-EV 근사이며 풀 GTO 균형이 아닙니다.** 상대가 최적 대응하는 균형점을
  푸는 것이 아니라, 고정된 휴리스틱 컨티뉴 레인지에 대한 기대값 계산입니다.
- 단일 콜러 모델: 멀티웨이 팟, 3벳/4벳 서브트리, 포지션별 개별 수비
  레인지가 없습니다. 이 때문에 실제 솔버 대비 일부 포지션 레인지가 넓거나
  좁게 나올 수 있습니다.
- 에퀴티 실현율·컨티뉴 % 는 손으로 캘리브레이션한 상수입니다.
- 몬테카를로 표본 오차가 있어 EV ≈ 0 경계 핸드의 오픈/폴드 판정은
  실행 설정(시드·반복 수)에 민감합니다. ICM·레이크·앤티 구조 미반영.

## 3. 포스트플랍 — 실시간 자체 MCCFR

- **위치**: `packages/engine/src/cfr.ts`
- **내용**: 리버 스팟을 MCCFR(Monte-Carlo Counterfactual Regret Minimization,
  external sampling)로 **요청 시 실시간 계산**합니다. 미리 계산된 포스트플랍
  데이터베이스는 없습니다.
- **한계**: 단순화된 싱글벳 리버 트리(체크/벳/콜/폴드)만 다루며, 반복 횟수가
  유한하므로 내시 균형으로 "수렴 중"인 근사 전략입니다. 플랍/턴 전체 트리는
  범위 밖입니다.

## 4. 푸시/폴드 — Nash 근사 자체 계산

- **위치**: `packages/engine/src/preflop.ts` (`pushFoldAdvice`),
  `packages/engine/src/spotev.ts` (`shoveEv`)
- **내용**: 숏스택 푸시/폴드는 (a) Chen 강도 + 스택·인원수 기반의 Nash 차트
  **근사 휴리스틱**, (b) 폴드 에퀴티와 콜 레인지 대비 에퀴티를 결합한 자체
  몬테카를로 칩-EV 모델 두 가지로 제공합니다.
- **한계**: 공식 Nash 균형표를 내장한 것이 아니라 그 형태를 근사한 것입니다.

## 5. ICM / MTT — 수학 공식 + 대표 상금 구조

- **위치**: `packages/engine/src/icm.ts`, `packages/engine/src/mtt.ts`
- **내용**: ICM은 표준 **Malmuth-Harville 재귀 공식**의 직접 구현(시뮬레이션
  아님, 결정론적 수학 계산)입니다. MTT 헬퍼는 **대표적인 표준 상금 구조
  프리셋**(자체 정의, 특정 대회 데이터 아님)과 버블 팩터, 단순화를 명시한
  ICM 쇼브 EV 모델을 제공합니다.
- **한계**: Malmuth-Harville 자체가 근사 모델이며(스택 크기 편향),
  상금 구조 프리셋은 "대표값"일 뿐 실제 대회별 구조와 다를 수 있습니다.

## 6. 외부 상용 데이터 미사용

- GTO Wizard, PioSOLVER, MonkerSolver 등 **상용 솔버의 출력물·데이터베이스·
  차트 파일을 복제하거나 포함하지 않았습니다.** 라이선스가 필요한 외부
  데이터셋도 사용하지 않습니다.
- 이 저장소의 모든 수치는 위 1–5의 자체 코드로 계산되거나 수작업으로 근사해
  작성된 것이며, 따라서 상용 솔버 수준의 정밀도를 보장하지 않습니다. 학습·참고
  용도로 사용하시고, 고액 실전 의사결정의 유일한 근거로 삼지 마세요.
